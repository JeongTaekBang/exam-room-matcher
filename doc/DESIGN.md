# 설계 문서: 강의실 배정 프로그램 (v2)

## 1. 개요

학과별 시험 요청사항과 기존 강의실 시간표를 **시각화**하여
사람이 수동으로 배정할 때 빠르게 판단할 수 있도록 돕는 도구.

**핵심 원칙:**
- 자동 배정 없음 — 모든 배정 판단은 사람이 한다
- 자동 해제 있음 — 미실시/부분교시/이동 원래강의실은 시스템이 해제
- 요청사항 텍스트는 원문 그대로 표시 — 자동 파싱/태그 없음
- 도구의 역할: 정보 시각화 + 충돌 감지 + 빈 강의실 검색 + 자동 해제

## 2. 입력 파일

| 파일 | 설명 |
|------|------|
| 요청사항 엑셀 | 학과별 시험 요청 (과목구분=공통만 대상) |
| 타임테이블 엑셀 | 기존 강의실 시간표 (5일 x 119실 x 0~14교시) |

폴더 구조: `{년도}/{학기}_{시험종류}/` (예: `2026/1학기_중간고사/`)

## 3. 아키텍처

```
data_loader.py          dashboard.py
┌──────────────┐       ┌──────────────────────┐
│ 엑셀 파싱     │──────→│ Streamlit 대시보드     │
│ 요청 분류     │       │ (5탭, 읽기+배정)       │
```

### data_loader.py — 데이터 계층

| 함수 | 역할 |
|------|------|
| `parse_schedule(raw)` | 수업시간표 문자열 → ScheduleSlot 리스트 |
| `load_requests(file)` | 요청사항 엑셀 → ExamRequest 리스트 |
| `classify_requests(reqs)` | 5개 카테고리 분류 |
| `load_timetable(file)` | 타임테이블 → (room_to_row, room_capacity, timetable_data) |
| `load_all(req, tt)` | 통합 진입점 |

### dashboard.py — UI 계층

5탭 (업무 흐름 순서):

| 순서 | 탭 | 역할 |
|------|-----|------|
| 1 | 기존 시간표 | 원본 시간표 격자 (갈색=수업, 청록=특수) |
| 2 | 배정 작업 | 진행 요약 → 필터/검색 → 히트맵+수급현황 → 미배정(이동/분반/기존유지) → 해제 → 점유현황 격자 → 작업현황 |
| 3 | 배정 현황 | 진행 요약 → 필터/검색 → 검수 큐 → 완료 목록 → 내보내기/이력 |
| 4 | 결과 검증 | 요청+배정 오버레이 시간표 + 충돌 감지 |
| 5 | 통계 | 분류별/일별/가동률 |

## 4. 분류 기준 (5개 카테고리)

N열(강의실선택) + K열(시험일자) 기반:

| 카테고리 | 조건 | 색상 |
|----------|------|------|
| NORMAL_EXAM | 유효 시험일자 + 기존강의실 | 파란 #4472C4 |
| NO_EXAM | 미실시/대체과제 키워드 | 핑크 #FF69B4 |
| ROOM_CHANGE | "강의실 변경 요청" + 유효일자 | 골드 #FFD700 |
| ROOM_SPLIT | "강의실 분반 요청" + 유효일자 | 보라 #9370DB |
| SKIP | 정보 불충분 | 회색 #AAAAAA |

요청사항(O열) 텍스트는 분류에 관여하지 않음 (NO_EXAM 키워드 매칭 제외).
수업시간표(I열)에서 시험일 요일 슬롯의 강의실을 `req.room`으로 자동 설정.

## 5. 충돌 감지

결과 검증 탭에서 같은 강의실+교시에 2개 이상 과목이 매핑되면:
- 빨간 셀 (#FF0000) + "충돌! N건" 텍스트
- tooltip에 겹치는 과목 전체 목록 + 요청사항 원문
- 상단 경고 배너

수동 배정 결과(session_state)도 충돌 감지에 포함.
`keep_orig=False`인 분반은 원래 강의실을 충돌 집계에서 제외.
같은 과목의 분반 배정끼리(`base_key` 동일)는 충돌로 판정하지 않음.

## 6. 잔여 슬롯 히트맵

요청 + 배정 탭 내 접을 수 있는 expander에 표시:
- 행: 수용인원 구간 (전체, 30+, 50+, 80+, 100+)
- 열: 0~14교시
- 셀값: 빈 강의실 수
- 색상: 빨강(0~3) → 노랑(4~10) → 초록(11+)

## 7. 클릭 배정 (session_state)

```python
st.session_state.assignments = {
    "과목명-분반": {
        "room": "N212",
        "sheet": "4.21.(화)",
        "periods": [4, 5],
        "original_room": "K106",
        "students": 60,
        "category": "강의실 변경",  # 또는 "강의실 분반", "시험 진행"
        "keep_orig": True,         # 분반 시 기존 강의실 유지 여부
    },
    "과목명-분반+2": { ... },      # 분반 추가 배정
}
```

- 배정 모드: 이동 / 분반(기존 유지+추가, 기존 미유지+추가) / 기존 강의실 유지
- 분반 다중 배정: 기본키, +2, +3 ... 순서로 키 생성 (`_next_split_key`)
- JSON 파일(`_assignments.json`)에 원자적 저장(tempfile→rename), 브라우저 새로고침 시 복원
- CSV/엑셀 다운로드로 외부 보존
- 이동 모드: 수용인원 < 수강생 시 배정 차단
- 분반 완료 판정: `keep_orig=True` → 1건 이상, `keep_orig=False` → 수용인원 합 ≥ 수강생
- 개별 배정 취소 가능 (배정 작업 현황 expander)

## 7-1. 자동 해제

| 유형 | 조건 | 라벨 | 색상 |
|------|------|------|------|
| NO_EXAM | 미실시/대체과제 분류 | 미실시 | 핑크 |
| ROOM_CHANGE | 원래 강의실 ≠ 배정 강의실 | 이동 | 핑크 |
| NORMAL_EXAM | 시험교시(L,M열) < 수업교시(I열 슬롯) | 부분해제 | 핑크 |

모두 구조화된 숫자 컬럼 기반 (요청사항 텍스트 미참조).
해제된 슬롯은 빈 강의실 검색/히트맵에 자동 반영.

## 8. 매핑 테이블

- 요일→시트: 화=4.21, 수=4.22, 목=4.23, 금=4.24, 월=4.27
- 교시→열: 0교시=F(col 6) ~ 14교시=T(col 20)
- 수업시간표 파싱: `([월화수목금토일])(\d+)(?:~(\d+))?\(([^)]*)\)`
- timetable_data: `{sheet: {room: {period: (value, color_rgb)}}}`

## 9. 의존성

- Python 3.8+
- openpyxl, streamlit, pandas
